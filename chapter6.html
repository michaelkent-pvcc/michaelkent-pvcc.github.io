Chapter 6: Programming Languages

Term

Definition

Machine language

A low-level programming language that uses binary code to communicate directly with a computer's hardware.

Debugging

The process of finding and fixing errors or bugs in a computer program.

Assembler

A program that translates assembly language code into machine language code.

Assembly language

A low-level programming language that uses symbolic representations of machine language instructions to communicate directly with a computer's hardware.

Translator

A program that translates source code from one programming language to another.

Compiler

A program that translates source code into machine language code.

Interpreter

A program that executes source code line by line without first compiling it into machine language code.

Cross-platform software

Software that can run on multiple operating systems or platforms.

Natural language

Human language, such as English or Spanish, that is used for communication.

Formal language

A precise and rigorous language used in mathematics, logic, and computer science to express algorithms, syntax rules, and other formal specifications.

Machine independence

The ability of a program or algorithm to work on different types of computers or devices without modification.

High-level programming language

A programming language that is further away from machine language and focuses on abstract concepts rather than low-level details of hardware operation.

Object-oriented programming (OOP)

A programming paradigm that organizes software design around objects, which have properties and methods that describe their behavior.

Scripting language

A programming language that supports scripts, which are small programs that automate tasks and are often executed at runtime.

Data types (integer, float, and string)

Primitive data types that represent integers, floating-point numbers, and text strings respectively.

Primitive data type

A basic data type that is not derived from other data types, such as integer, float, or boolean.

Data structure

A format for organizing and storing data in a computer program, such as arrays, linked lists, trees, or graphs.

Python collection data structures (list, tuple, set, and dictionary)

Built-in Python data structures that allow efficient storage and manipulation of data, including lists, tuples, sets, and dictionaries.

Function

A block of code that performs a specific task and can be called repeatedly from different parts of a program.

Local variable

A variable that has a limited scope and is only accessible within a particular function or block of code.

Global variable

A variable that can be accessed from any part of a program.

Source program

The original version of a program written in a high-level programming language.

Object program

The compiled or assembled version of a program that can be executed directly by a computer's hardware.
